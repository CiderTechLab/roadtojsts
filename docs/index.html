JavaScript

概要

JavaScript（以下、JS）は当時Netscape社のブレンダン・アイク氏によって開発され、1995年に初めてのバージョンがリリースされました。当時は簡単な用途を想定されており、「ブラウザで動作するJava風のプログラミング言語作ってくれ。（意訳）」と上司に言われ、なんと10日で作成されたと言われています。ちなみに、後にブレンダン・アイク氏はMozillaのCEOも経験し、現在ではBraveブラウザの開発にも携わっています。

TypeScript

歴史

TypeScript（以下、TS）はMicrosoftによって開発され、2012年に初めてのバージョンがリリースされました。

コンパイル設定

tsconfig.json

tsconfig.jsonは、TypeScriptのコンパイル設定を記述する重要なファイルです。以下、記述例です。 

{"compilerOptions": {"target": "es2015"}}

通常、TSのコンパイルは設定しないと互換性維持のためデフォルトでES5にコンパイルされます。しかし、上記の記述例でコンパイル後のJSのバージョンをECMAScript2015（ES6）に設定することができます。

設定一覧

設定名 説明
target コンパイル後のJavaScriptのバージョンを指定します。

"outDir": "dist",

この設定により、コンパイル後のJavaScriptファイルがdistディレクトリに出力されます。

Tailwind CSS

概要

Tailwind CSSはOSSのCSSフレームワークです。特徴としてユーティリティファーストが挙げられます。他のCSSフレームワークは、ボタンやボックスなど既に定義済みのコンポーネントを使い制作していきますが、Tailwind CSSはユーティリティークラスという特定のスタイルが設定されているクラスをHTML要素に組み合わせて適用することで、自由度が高く保守性が高いコーディングを実現しています。

導入方法

私は当初、Tailwind CSSの公式サイトを参考に以下のコードを<head>要素内に記述し、試験的に導入しておりました。 

<script src="https://cdn.tailwindcss.com"></script>

その後「同じ要素に、毎回同じTailwind CSSのクラスを記述するのは冗長だな」と感じ、npmを経由し本格導入することにしました。

ビルド～読み込み方法

docs>src 内にあるTailwind CSSの構文で制作したCSSファイルを以下のコマンドでbuildし、docsフォルダ内に生成します。 

npm run build

その後、生成したCSSファイルをhtml側で読み込みます。
