制作の流れ

このサイトの構築プロセス

RoadToTypeScriptは、生成AIの支援を受けながら段階的に構築されました。以下は制作の基本的なフローです。

  1. VSCodeのサイドバーチャットにプロンプトを記入

    実装したい機能や修正内容を、自然言語でチャットに入力します。例えば、「目次を自動生成したい」「レスポンシブレイアウトにしてほしい」といった内容です。

  2. 追加・編集されたコードを確認し、「保持」ボタンをクリック

    AIが生成したコード変更をVSCode上で確認します。意図通りであれば、「保持」ボタンをクリックして変更を確定させます。

  3. 見た目をブラウザで確認

    変更内容がブラウザでどのように表示されるかを確認します。レイアウト、色、フォント、レスポンシブ対応など、デザイン面での問題がないかチェックします。

  4. 修正点があれば最初に戻る

    問題が見つかった場合は、プロンプトの記入に戻り、修正内容をプロンプトとして入力します。この反復プロセスを繰り返すことで、品質を高めていきます。

まとめ

この段階的で反復的なアプローチにより、機能追加とデザイン調整を効率的に進めることができます。

学習方針

  • 毎日コードを手で書く（src/ディレクトリに.tsファイルを作成）

TypeScript 学習メモ

tsconfig.json

TypeScriptのコンパイル設定を記述するファイルです。


{
  "compilerOptions": {
    "target": "es2015"
  }
}

上記で、コンパイル後のJavaScriptのバージョンをECMAScript2015に指定できます。 指定しないとデフォルトでES5になります。

"outDir": "dist",

この設定により、コンパイル後のJavaScriptファイルがdistディレクトリに出力されます。

TypeScriptの型一覧

型名 使用例 型の概要
string let name: string = "Alice"; 文字列型
number let age: number = 25; 数値型（整数、小数、Infinity、NaN）
boolean let isActive: boolean = true; 真偽値型（trueまたはfalse）
any let value: any = "anything"; 任意の型。型チェックを無効化
unknown let value: unknown = "anything"; 任意の型だが、使用前に型を絞る必要がある
void function log(): void { } 値を返さない関数の戻り値型
null let value: null = null; nullを表す型
undefined let value: undefined; undefinedを表す型
never function error(): never { throw Error(); } 値を返さない（エラーまたは無限ループ）
object let obj: object = { }; オブジェクト型（プリミティブ以外）
Type[] let nums: number[] = [1, 2, 3]; 配列型
Union let value: string | number; 複数の型のいずれか
Literal let direction: "up" | "down"; 特定の値に限定する型
Tuple let tuple: [string, number]; 固定長の配列で要素ごとに型を指定
enum enum Color { Red, Green, Blue } 名前付き定数の集合

Q.ではTypeScriptで型宣言は必須なのか？

  • A.型推論により、必ずしも明示的な型宣言は必要ないです！（const variable = "text"; のように）
  • ただし、関数の引数・戻り値、データベースでは明示的な型宣言が推奨されます。

TailWind css

このページではTailWind cssを試験的に導入しております。


<script src="https://cdn.tailwindcss.com"></script>

GitHub IOでTailWind cssを使う

1. ローカル環境でセットアップ

npm init -y
  npm install -D tailwindcss postcss autoprefixer
  npx tailwindcss init -p

2. tailwind.config.js を修正

/**
 * @type {import('tailwindcss').Config}
 */
export default {
  content: [
    "./docs/**/*.{html,js}",
  ],
  theme: {
    extend: {},
  },
  plugins: [],
}

3. input.css を作成

@tailwind base;
@tailwind components;
@tailwind utilities;

4. package.json にビルドスクリプト追加

{
  "scripts": {
    "build": "tailwindcss -i ./input.css -o ./docs/output.css"
  }
}

5. GitHub Actions ワークフロー作成

name: Build and Deploy

on:
  push:
    branches: [main]

jobs:
  build:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v3
      - uses: actions/setup-node@v3
        with:
          node-version: '18'
      - run: npm install
      - run: npm run build
      - uses: actions/upload-pages-artifact@v2
        with:
          path: './docs'

  deploy:
    needs: build
    permissions:
      pages: write
      id-token: write
    environment:
      name: github-pages
      url: ${{ steps.deployment.outputs.page_url }}
    runs-on: ubuntu-latest
    steps:
      - name: Deploy to GitHub Pages
        id: deployment
        uses: actions/deploy-pages@v2

6. index.htmlを修正

<head>
  <meta charset="UTF-8" />
  <meta name="viewport" content="width=device-width, initial-scale=1.0" />
  <link rel="stylesheet" href="output.css" />
  <link rel="stylesheet" href="style.css" />
  <title>RoadToTypeScript</title>
</head>

CDNスクリプトを削除して、生成されたoutput.cssを読み込みます。

7. GitHub設定

GitHub リポジトリ > Settings > Pages で以下を選択：

  • Source: GitHub Actions

これで、mainブランチにpushするたびに自動的にTailwindがビルドされ、デプロイされます。
